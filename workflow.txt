# Debian Docker workflow

Run the first block from macOS in the project directory. Run the second block
only after the prompt changes to `root@<container>:/work#`.

## 1. Build and start the Debian container from macOS

##bash
colima start

docker build \
  --platform linux/arm64 \
  -t flow-anonymizer:debian \
  .

docker run --rm -it \
  --platform linux/arm64 \
  -v "$PWD":/work \
  -w /work \
  flow-anonymizer:debian
##

The multi-stage Docker build uses Debian Bookworm. It compiles the official
nfdump v1.7.3 `nfgen` test utility in the builder stage and copies only the
resulting binary into the runtime image. The runtime installs `nfdump`, which
also supplies `nfanon` and `nfcapd`.

## 2. Run the complete workflow inside Docker

##bash
# Confirm Debian and all required flow tools.
cat /etc/os-release
nfdump -V
command -v nfanon
command -v nfcapd
command -v nfgen
file "$(command -v nfgen)"

# Ensure the project scripts are executable.
chmod +x *.sh

# Create or reuse the local-only key and synthetic nfgen input.
./generate_key.sh
./make_sample_data.sh

# Inspect the generated nfdump file.
RAW_FILE="raw/2026-01/2026-01-01/nfcapd.202601010000"
test -f "$RAW_FILE" || { echo "ERROR: sample file was not generated"; exit 1; }
nfdump -r "$RAW_FILE" -o extended | head
nfdump -q -N -r "$RAW_FILE" -o 'fmt:%cnt' |
  awk 'NF { count++ } END { print "Readable records:", count + 0 }'

# Preview, execute, and validate anonymization.
./anonymize_flows.sh
./anonymize_flows.sh --run
find anon -type f -name 'nfcapd.*' -print
./validate_flows.sh
cat logs/validation.txt
##

Successful validation ends with:

##text
files checked: 1/1
OVERALL: PASS — required utility preserved and all IP fields pseudonymized
##

## 3. Exit

##bash
exit
##

The container is removed because it uses `--rm`. The bind-mounted project,
including ignored local keys, sample files, anonymized output, and logs,
remains on the Mac.

## Output files

- `raw/2026-01/2026-01-01/nfcapd.202601010000` — synthetic input.
- `anon/2026-01/2026-01-01/nfcapd.202601010000` — anonymized output.
- `logs/anonymize.log` — input/output audit trail and key fingerprint.
- `logs/validation.txt` — complete validation results.
- `secret/anon.key` — local secret; never print, share, or commit it.

Outside Docker, verify repository hygiene with:

##bash
git status --ignored
##

The `secret/`, `raw/`, `anon/`, `logs/`, and `.local-tools/` paths must remain
ignored.
